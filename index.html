Method

Quality-aware verification

1

Embed both faces

ID photos and selfies are aligned through DeepFace and encoded with published face-recognition backbones such as ArcFace and FaceNet512.

2

Measure visual quality

Classic computer-vision signals estimate blur using variance of Laplacian, exposure using grayscale brightness, and face visibility using detector success.

3

Fuse similarity and confidence

The final score combines embedding similarity with pair-level image quality, making the output easier to interpret in difficult lighting or blurry-image cases.

Computer Vision Contribution

Beyond “just use ArcFace”

The published face-recognition model provides the identity signal. Face Bridge adds a transparent quality layer around it, so the system can account for common ID-to-selfie failure modes such as soft focus, underexposure, overexposure, and poor face visibility.

overall quality = 0.35 brightness + 0.35 sharpness + 0.30 face visibility

final score = 0.80 similarity + 0.20 pair quality

Ablation

Two-domain adapter prototype

A second experiment trained separate residual adapter heads for ID photos and selfies. On the small public sample, this was a useful negative result: the adapters did no harm but did not outperform the frozen ArcFace baseline.

Poster Narrative

What to emphasize

Problem

ID photos and selfies differ in pose, lighting, capture device, compression, and image age.

Approach

Use a strong face-recognition backbone, then calibrate confidence with blur, exposure, and face visibility.

Main figure

Show the ROC curve comparing ArcFace cosine similarity against quality-aware fusion.

Claim

The project adds an interpretable quality-aware layer to a highly cited face-verification baseline.

Reproducibility

Code, data, and privacy

The public repository includes source code, aggregate metrics, and plots. Raw biometric images, ID documents, Kaggle credentials, model weights, local embedding caches, and pair-level path CSVs are excluded.
